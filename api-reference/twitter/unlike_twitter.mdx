Unlike a tweet on behalf of the authenticated source user. Routed through the AIsa gateway at `https://api.aisa.one/apis/v1/twitter/unlike_twitter`.

## Prerequisites

- An **AIsa API key**. Pass it as `aisa_api_key` in the request body.
- A one-time **OAuth authorization** for the source user account. Link your X account by calling [`POST /apis/v1/twitter/auth_twitter`](/api-reference/twitter/auth_twitter) — AIsa stores the session against your API key and uses it automatically on every write call.
- The X session must hold `like.write`, `tweet.read`, `users.read`.

## Request body

| Field | Type | Required | Meaning |
| --- | --- | --- | --- |
| `aisa_api_key` | string | yes | Your AIsa API key. |
| `tweet_id` | string | yes | Numeric ID of the tweet to unlike. |

## Response

```json
{
  "ok": true,
  "code": 200,
  "data": {
    "liked": false
  }
}
```

**`data.liked`** — `false` once the like has been removed.

## Idempotency

Unliking a tweet the source user has not liked is a no-op. The endpoint still returns `200` with `{ "ok": true, "code": 200, "data": { "liked": false } }`. Safe to retry.

## Common 4xx causes

- `400 invalid-request` — missing or non-numeric `tweet_id`.
- `403 client-forbidden` — OAuth session missing the `like.write` scope. Re-link via [`auth_twitter`](/api-reference/twitter/auth_twitter).
- `404 resource-not-found` — target not found or not visible to the source user.

See [Error Codes](/api-reference/errors) and [Rate Limits](/api-reference/rate-limits) for more.
