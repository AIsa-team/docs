description: "Start the OAuth flow that links an X/Twitter account to your AIsa API key. Call once per source user before using any Twitter write endpoint (follow, like, post, DM)."

`POST /apis/v1/twitter/auth_twitter` is the first step for any X/Twitter **write** action routed through AIsa. It returns a short-lived X OAuth authorization URL — the source user opens it in a browser, approves the requested scopes, and X redirects back to AIsa's fixed callback. AIsa stores the resulting session against your API key, and every subsequent write call (e.g., [`POST /twitter/follow_twitter`](/api-reference/twitter/follow_twitter)) uses that session automatically.

## When to call it

- **Once per source user**, the first time you link their X account.
- **Again** if the stored session is revoked, expired, or you need a different scope set.

You do not call it before every write request. The OAuth session persists against your AIsa API key.

## Flow

<Steps>
  <Step title="Start the flow">
    Send `POST /apis/v1/twitter/auth_twitter` with `{ "aisa_api_key": "..." }` in the body.
  </Step>
  <Step title="Open the auth URL">
    The response returns `{ "ok": true, "code": 200, "data": { "auth_url": "..." } }`. Open `data.auth_url` in the source user's browser. X shows its standard "Authorize AIsa to access your account" screen.
  </Step>
  <Step title="User approves">
    After the user clicks Authorize, X redirects to AIsa's fixed callback (`https://api.aisa.one/apis/v1/twitter/oauth_callback`). AIsa exchanges the authorization code for tokens and stores the session against your `aisa_api_key`.
  </Step>
  <Step title="Write endpoints unlock">
    From this point on, any Twitter write endpoint called with the same `aisa_api_key` acts on behalf of the linked source user. You never pass OAuth tokens to those calls.
  </Step>
</Steps>

## Scopes

AIsa requests the full set needed for every Twitter write endpoint we expose:

- `follows.write` — [follow](/api-reference/twitter/follow_twitter) / [unfollow](/api-reference/twitter/unfollow_twitter)
- `tweet.read` — read tweets (required by most write actions)
- `users.read` — resolve users, read profile info
- `tweet.write` — [post](/api-reference/twitter/post_twitter), reply, quote
- `like.write` — [like](/api-reference/twitter/like_twitter) / [unlike](/api-reference/twitter/unlike_twitter)

Write calls that require a missing scope return `403`.

## Re-linking

If the stored session is revoked (user clicks "Revoke access" on X's app settings) or expires, the next write call returns `403`. Re-run `POST /apis/v1/twitter/auth_twitter` to generate a new `auth_url` and restore the session.
